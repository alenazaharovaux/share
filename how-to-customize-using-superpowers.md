# Как generic скилл using-superpowers был дополнен под конкретные проекты

## Что такое using-superpowers

Скилл из пакета superpowers — «мета-диспетчер», который заставляет Claude проверять применимые скиллы перед каждым действием. Содержит общий принцип («проверь скилл до того, как что-то делать»), таблицу рационализаций и приоритет вызова: сначала процессные скиллы, потом доменные.

Скилл работает из коробки и обновляется автоматически. Менять его напрямую нет смысла — перезапишется при обновлении.

## Почему потребовалось дополнение

Скилл написан для универсального разработчика: предполагает, что основная работа — код, баги, фичи. Но если профиль задач шире, generic скилл начинает пропускать целые категории работы:

- **Исследовательские проекты** — анализ интервью, транскрипты встреч, кейсы, количественные данные
- **Редакционный конвейер** — статьи проходят через цепочку от идеи до публикации
- **Веб-разработка** — здесь скилл работает как задумано
- **Python-боты и автоматизация** — тоже стандартная разработка
- **Obsidian vault** — организация знаний, заметки, не код вовсе

Generic скилл говорит «проверь, есть ли подходящий скилл», но не подсказывает Claude, **какой именно** скилл вызывать для исследовательской задачи или для написания статьи. В результате журналистские и аналитические скиллы (data-journalism, ai-writing-detox, newsroom-style) вызывались нерегулярно — Claude не всегда связывал «анализ транскрипта» с data-journalism.

## Что было сделано

В глобальный CLAUDE.md добавлен **явный маппинг задач на скиллы** — конкретная таблица «если задача такая, вызывай этот скилл». Generic скилл даёт принцип, CLAUDE.md даёт конкретику:

| Generic скилл говорит | CLAUDE.md уточняет |
|---|---|
| «Проверь скиллы перед действием» | Конкретный список: brainstorming -> writing-plans -> executing-plans, и так для каждого типа задач |
| «Process skills first» | Кодовые слова-триггеры на русском: «скилл?», «стоп», «план?», «проверь» |
| «Don't rationalize» | Явные ловушки: слово «implement» в промпте ≠ разрешение пропустить согласование |

Последнее дополнение — два маппинга, которых не хватало:

- **Анализ интервью / транскриптов / данных исследований** -> `data-journalism`
- **Написание текстов / статей / постов** -> `ai-writing-detox` + `newsroom-style`

## Итоговая архитектура

```
using-superpowers (generic, обновляется автоматически)
    └── «всегда проверяй скиллы, не рационализируй»

CLAUDE.md (кастомный, не перезаписывается)
    └── конкретный маппинг под проекты:
        - код/фичи → brainstorming, frontend-design, TDD
        - баги → systematic-debugging
        - исследования → data-journalism
        - тексты → ai-writing-detox + newsroom-style
        - планы → writing-plans → executing-plans
        - триггеры на родном языке пользователя
```

Generic скилл задаёт дисциплину, CLAUDE.md направляет её в нужное русло. Один без другого работает хуже: скилл без маппинга пропускает нестандартные задачи, маппинг без скилла не защищает от рационализаций «это же простая задача, скилл не нужен».

## Важная оговорка

Эта кастомизация привязана к конкретному набору проектов, в котором исследования и редактура занимают значительную долю наравне с разработкой. Для чисто кодовых проектов generic скилл может быть достаточен сам по себе, без дополнительного маппинга в CLAUDE.md.
